# Devlog: 11 марта 2026 (среда)

## Цель

Реализация прототипа API-сервисов на фреймворке userver для обеспечения функций
отслеживания и расчета стоимости доставки.

## Контекст

Компания оказывает услуги доставки. Требуется реализовать два публичных сервиса:
отслеживание отправлений по трек-номеру и расчет стоимости доставки. Оба сервиса должны
быть доступны как через сайт, так и через чат-бота, а в перспективе — напрямую клиентам
компании для интеграций.

## Архитектурные решения  

Чтобы избежать дублирование логики для сайта и бота, оба сервиса (отслеживание и расчет
стоимости) решено реализовать как отдельные компоненты (микросервисы) с единым REST API.

### Политика безопасности и доступа

Данные: Сервис оперирует только публичными данными. Любой человек может получить
информацию по любому трек-номеру или воспользоваться калькулятором стоимости.

Аутентификация: Для идентификации клиента используется постоянный API-ключ. Ключ
передается в запросе (например, в заголовке X-API-Key).

Авторизация: Для ограничения доступа к служебным конечным точкам с ключом связывается
список scopes к которым владельцу ключа дается доступ.

Защита от перегрузок: На первом этапе - средствами front nginx и блокировкой/удалением
ключа. В перспективе с каждым ключом можно будет связать или лимиты или уровень сервиса и
реализовать собственные методы ограничения нагрузки.
